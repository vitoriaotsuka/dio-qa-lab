# 🧪 QA Test Challenge - Documentação de Testes

Este repositório apresenta a documentação desenvolvida durante um desafio prático de Quality Assurance (QA) - DIO, com foco na validação da funcionalidade de autenticação de usuários em uma loja virtual.

---

## 🎯 Objetivo

Demonstrar conhecimentos fundamentais em QA, incluindo:

* Definição de fluxo de trabalho de desenvolvimento
* Ciclo de vida do bug
* Escrita de User Stories
* Elaboração de casos de teste
* Aplicação de BDD (Gherkin)

---

## 📂 Estrutura do Projeto

Os documentos estão organizados na pasta `docs/`:

* 📄 **Fluxo de trabalho e ciclo de vida do bug**
* 📄 **User Stories**
* 📄 **Casos de teste (step-by-step e BDD)**
* 🧠 **Mind map da funcionalidade de login**

## 🔐 Escopo do Projeto

A funcionalidade analisada neste projeto é:

* Autenticação de usuário (Login)
* Recuperação de senha

---

## 🔄 Fluxo de Trabalho

Backlog → To Do → In Progress → Code Review → Test (QA) → Done

---

## 🐞 Ciclo de Vida do Bug

New → Assigned → In Progress → Fixed → Retest → Closed
Reopened → In Progress

---

## 🧪 Casos de Teste

Foram elaborados casos de teste para validar:

* Login com credenciais válidas
* Login com credenciais inválidas

---

## 🥒 BDD (Gherkin)

Foram definidos cenários para:

* Login com sucesso
* Login inválido

---

## 🛠️ Ferramentas Utilizadas

* Miro
* Gherkin
* Git e GitHub
